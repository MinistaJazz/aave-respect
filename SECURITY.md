# Security And Safety

This project is a public standard and skill, not a hosted service. Security here includes both technical issues and safety issues that could expose or harm AAVE speakers.

## Report Privately

Email `legal@muchdifferentworld.com` for sensitive issues.

Use private reporting for:

- unconsented speech, transcript, screenshot, or community material
- examples that identify a private person
- misuse by a company claiming certification or endorsement
- implementation behavior that could create legal, clinical, educational, employment, or safety harm
- prompt or benchmark issues that should not be publicly disclosed before review
- repository security concerns

Do not put private data in a public issue.

## Public Issues

Public issues are appropriate for:

- documentation corrections
- non-sensitive benchmark suggestions
- general implementation questions
- public-source linguistic references

If you are unsure whether something is sensitive, report it privately first.

## What To Include

When reporting a safety issue, include:

- what happened
- where it happened
- why it creates harm or misuse risk
- whether private people or private data are involved
- links or screenshots only if you have the right to share them

## Response

Maintainers will prioritize reports involving consent, identity exposure, false certification claims, or high-stakes domains such as legal, medical, educational, employment, and public benefits systems.

Not every harmful implementation can be fixed from this repository, but we can update guidance, benchmarks, warnings, and project policy when the standard needs stronger protection.
